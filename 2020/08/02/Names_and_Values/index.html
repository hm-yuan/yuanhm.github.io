Names and Values

对象（object）并没有名字（name），只是名字有了对象」

名字与对象的链接

名字和对象是独立的，二者通过“ <- ”相连。

  • 每个对象在R中都有自己的编号（如下的 “0x1deb7a5c530”）。
  • 名字与对象直接相连，名字只是对象的查询编码（Reference）：
# 变量与对象的关系
x <- c(1, 2, 3)
y <- x

# x, y实际指向了同一个对象
lobstr::obj_addr(x)
# [1] "0x1deba3bd8f8"
lobstr::obj_addr(y)
# [1] "0x1deba3bd8f8"

Copy-on-modify

  1. Basic：对象是不可修改的，只能对复制的对象进行修改（Copy-on-modify）
    • 修改y并不会改变x的值，只是通过复制对象，并在复本的基础上新生成了一个对象：
# 追踪x对象：如果复制发生，则将通知
tracemem(x)
#> [1] "<000001DEBA3BD8F8>"
y[[3]] <- 4L
#> tracemem[0x000001deba3bd8f8 -> 0x000001deba3b0d58]:

# 修改y并不会改变x的值
x
#> [1] 1 2 3
y
#> [1] 1 2 4

# 修改y实际上新生成了一个变量
lobstr::obj_addr(y)
#> [1] "0x1deba3b0d58"
  1. Function：函数（Function）中的对象同样使用Copy-on-modify准则
    • 函数的运行结果相当于建立了新的链接，其中没有涉及复制：
f <- function(a) a
x <- c(1, 2, 3)
tracemem(x)
z <- f(x)
# 没有复制发生！
  1. List：列表（list）指向的不是值（value），而是查询编码（Reference）的集合。
    • 当修改发生时，只是修改部分查询编码的指向：
l1 <- c(1, 2, 3)
l2 <- l1
l2[[3]] <- 4L
lobstr:ref(l1, l2)
#> o [1:0x1deb8aecdf0] <list> 
#> +-[2:0x1deba502740] <dbl> 
#> +-[3:0x1deba502778] <dbl> 
#> \-[4:0x1deba5027b0] <dbl> 
 
#> o [5:0x1deba41de78] <list> 
#> +-[2:0x1deba502740] 
#> +-[3:0x1deba502778] 
#> \-[6:0x1deba5f0fd0] <int>
  1. Date Frame：数据框（Data frames）是向量的列表（list of vectors），适用相同的规制。
    • 修改列时：仅有被修改的列指向的对象会发生改变。
    • 修改行时：所有列指向的对象都发生了改变。
  2. Character vectors：字符串向量（Character vectors）指向的是在全局中包含多种字符串的池。当修改发生时，是部分字符串的查询编码发生了改变，因而总体的对象也不同了
x <- c("a", "a", "abc", "d")
y <- x
y[[1]] <- "c"
# 指向的对象不同了
ref(x, y)
#> [1:0x1deb8b09248] <chr> 

#> [2:0x1deb8ab1b10] <chr> 

# 实际是具体修改的字符串查询编码变化了
ref(x, y, character = T)
#> o [1:0x1deb8b09248] <chr> 
#> +-[2:0x1deb133d720] <string: "a"> 
#> +-[2:0x1deb133d720] 
#> +-[3:0x1deb550c728] <string: "abc"> 
#> \-[4:0x1deb150d048] <string: "d"> 
 
#> o [5:0x1deb8ab1b10] <chr> 
#> +-[6:0x1deb00e29f0] <string: "c"> 
#> +-[2:0x1deb133d720] 
#> +-[3:0x1deb550c728] 
#> \-[4:0x1deb150d048]

对象的大小（Object size）

  1. 由于列表（list）实质上是储存了对象的查询编码，所以列表的大小可能远小于预期。
x <- runif(1e6)
lobstr::obj_size(x)
#> 8,000,048 B
y <- list(x, x, x)
lobstr::obj_size(y)
#> 8,000,128 B
  1. 由于R中的字符串同样采用类似于list的字符串池（string pool）的方式，所以重复字符串后的大小也小于预期。
banana <- "bananas bananas bananas"
obj_size(banana)
#> 136 B
obj_size(rep(banana, 100))
#> 928 B
  1. 变量相加后的大小类似于并集的关系，如果二者没有相同的值，那么对象相加后的大小就等于两个单独的大小相加。

立即修改

并非所有对对象的修改都是copy-on-modify，有两类例外，他们是将对象本身立即修改的（modify-in-place）：

  1. 仅有单一链接（binding）的对象

  1. 环境（environment）

    环境（environment）是一种特殊的对象，实质是多个对象和名称链接的集合，它总是立即修改的（modify-in-place）。

e1 <- rlang::env(a = 1, b = 2, c = 3)
e2 <- e1

# 修改e1中的对象后，e2同步改变了
e1$c <- 4
e2$c
#> [1] 4

清理内存

使用rm( )只能删去名称与对象的链接，并不能删去对象。

  • 使用gc( )来清理内存，gcinfo( TRUE )来显示每次计算机清理内存的操作。
  • 使用men_used( )来查看已消耗的内存。
